# Candidate Prep Deep Interview Design

**Дата:** 2026-07-17  
**Статус:** Затверджено в brainstorming  
**Scope:** Глибший збір інформації в prep-чаті Candidate Agent — follow-up питання, блок умов роботи, оновлений READY gate

**Передумови:** Contact bootstrap (2026-07-16), Candidate Agent Day 11, profile extraction/confirm

---

## Мета

Зробити prep-чат AI-кандидата максимально інформативним:

1. **Глибокі follow-up** — агент ставить уточнюючі питання доки відповідь не стане конкретною (цифри, назви, приклади, терміни), а також заглиблюється в конкретні згадки кандидата (технології, проєкти, компанії, метрики).
2. **Блок умов роботи** — збір базових очікувань: формат, зарплата, графік, релокація.
3. **Збагачений профіль** — умови роботи зберігаються в існуючому полі `goals` без зміни схеми БД; live Candidate Agent автоматично отримує більше контексту для відповідей і питань компанії.

---

## Узгоджені рішення (brainstorming)

| Питання | Рішення |
|---------|---------|
| Глибина follow-up | Доки відповідь не стане конкретною; без жорсткого ліміту раундів на тему |
| Умови роботи | Базовий набір: формат, зарплата, графік, релокація |
| Зберігання | Розширити поле `goals`; схема БД не змінюється |
| Контекст вакансії | Ні — універсальні питання |
| READY gate | Усі теми зібрані; відмова після уточнення = `не вказано` (як телефон) |
| Підхід | Prompt-only (A); серверна перевірка READY — non-goal на цьому етапі |

## Структура діалогу

Після contact bootstrap (без змін) агент веде **5 тем** по черзі, одне питання за раз:

1. **Досвід** — ролі, роки, проєкти, технології, масштаб.
2. **Сильні сторони** — навички з прикладами та результатами.
3. **Зони росту** — конструктивно, без токсичної самокритики.
4. **Умови роботи** *(новий блок)* — формат, зарплата, графік, релокація.
5. **Кар'єрні цілі** — напрямок розвитку, тип продукту/команди, горизонт.

Агент не показує кандидату весь список тем одразу — веде природну розмову.

### Правила follow-up

Замінити правило «одне коротке уточнювальне питання» на:

- Якщо відповідь загальна («працював з базами даних», «хочу розвиватися») — ставити уточнюючі питання **доки не з'явиться конкретика**: роки, назви технологій, цифри, приклади, терміни.
- Якщо кандидат згадав щось конкретне (компанію, технологію, проєкт, метрику, роль) — **обов'язково** заглибитися: масштаб, роль, результат, контекст.
- Завжди **одне питання за раз**; кількість раундів на тему не обмежена.
- Не вигадувати факти за кандидата.

**Приклади для промпту (стиль, не шаблони):**

| Згадка кандидата | Можливий follow-up |
|------------------|-------------------|
| «FinTech» | Який продукт, який стек, яка роль |
| «Оптимізував БД» | Які запити, який був/став час відповіді |
| «Remote» | Повністю чи гібрид, скільки днів в офісі |
| «Вів команду» | Скільки людей, які обов'язки |

## Блок «Умови роботи»

Чотири підтеми по черзі:

| Підтема | Що збираємо | Приклад follow-up |
|---------|-------------|-------------------|
| Формат | офіс / гібрид / remote | «Скільки днів на тиждень готові бути в офісі?» |
| Зарплата | діапазон або мінімум (валюта) | «Це gross чи net?» — лише якщо кандидат сам назвав суму |
| Графік | повний день / part-time / гнучкий | «Чи є обмеження по годинах?» |
| Релокація | готовність / лише поточне місто | «Які міста або країни розглядаєте?» |

**Відмова:** одне коротке пояснення «навіщо це потрібно»; при повторній відмові — не блокувати, перейти до наступної підтеми. У профілі: `"<підтема>: не вказано"`.

**Кар'єрні цілі** (тема 5) — окремо від умов; ті самі правила глибоких follow-up.

## READY gate

`READY:true` лише коли **одночасно**:

- зібрано `fullName`;
- є доступний `email` (наданий кандидатом або fallback з акаунта);
- відбулося щонайменше 3 змістовні обміни після contact bootstrap;
- **Досвід** — конкретика (ролі, роки, проєкти/технології), не загальні фрази;
- **Сильні сторони** — з прикладами або результатами;
- **Зони росту** — хоча б одна конкретна зона;
- **Умови роботи** — усі 4 підтеми: конкретна відповідь або явна відмова після уточнення;
- **Кар'єрні цілі** — конкретика, не «хочу розвиватися».

**Телефон** — без змін: відсутність після повторного запиту з поясненням не блокує `READY:true`.

У всіх інших випадках — `READY:false`.

## Extraction: поле `goals`

Оновити `candidate-profile-extraction.uk.ts`: `goals` містить **і** умови роботи, **і** кар'єрні цілі з префіксами:

```json
"goals": [
  "Формат: повністю remote",
  "Зарплата: від 4000 USD gross",
  "Графік: повний день, гнучкий старт",
  "Релокація: не розглядає",
  "Ціль: перейти на senior у продуктовій FinTech-команді"
]
```

- Відмови → `"Зарплата: не вказано"` (аналогічно для інших підтем).
- Кар'єрні цілі — рядки з префіксом `"Ціль: ..."`.
- Якщо тема не обговорювалась — `["не вказано"]` для всього масиву (існуюче правило extraction).

Схема `CandidateProfile` і UI підтвердження **не змінюються**.

## Downstream-ефект

**Live-співбесіда:** Candidate Live Agent при `CANDIDATE_QUESTIONS` зможе ставити питання компанії про зарплату, формат, графік на основі `goals` — менше `needsHuman:true`, коли HR питає про очікування кандидата.

**Профіль:** після `finish` кандидат бачить структурований `goals` і може відредагувати перед confirm.

## Acceptance criteria

1. Prep-агент ставить follow-up доки відповідь не конкретна; заглиблюється в згадані кандидатом деталі.
2. Агент проходить блок умов роботи (4 підтеми) перед кар'єрними цілями.
3. `READY:true` лише при виконанні всіх критеріїв; відмова по підтемі не блокує.
4. Extraction записує умови і цілі в `goals` з префіксами.
5. Unit-тести оновлені; `npm run build` проходить.
6. `manual-test-dialogues.uk.md` оновлено для ручної перевірки.
