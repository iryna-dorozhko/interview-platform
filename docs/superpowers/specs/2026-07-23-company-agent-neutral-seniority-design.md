# Company Agent: нейтральний рівень посади — Design Spec

**Дата:** 2026-07-23  
**Статус:** Затверджено в brainstorming  
**Контекст:** HR prep-чат з Company Agent збирає профіль вакансії; вакансії можуть бути не лише з ІТ  
**Передумови:** `COMPANY_AGENT_SYSTEM_PROMPT_UK` у `backend/src/agents/prompts/company-agent.uk.ts`; тести в `company-agent.test.ts`  
**Мова:** Українська (повідомлення агента); код — англійською

---

## Проблема

Тема «Посада» у system prompt явно задає `рівень — junior/middle/senior`. Через це агент часто одразу питає про IT-грейд, що некоректно для не-ІТ вакансій і нав’язує зайве уточнення на першому кроці.

## Мета

1. Перше питання після привітання — лише назва посади / роль.
2. Не нав’язувати junior/middle/senior.
3. За потреби уточнювати досвід / грейд / рівень відповідальності нейтрально — і лише якщо HR сам цього не згадав.
4. Команду збирати наступним кроком після ролі (і за потреби після нейтрального уточнення).

## Рішення з brainstorming

| Питання | Рішення |
|---------|---------|
| Формулювання рівня | **3** — нейтрально (досвід / грейд / відповідальність), не junior/middle/senior |
| Порядок питань | **1** — спочатку тільки роль; рівень лише як уточнення, якщо бракує |
| Підхід реалізації | **A** — лише правка system prompt + тест |

## Поза scope

- Фіксований opening message у коді (підхід B)
- Нове поле seniority у схемі профілю / extraction (підхід C)
- Зміни UI, API, live-агентів
- Candidate Agent prompts

## Зміни

### 1. `backend/src/agents/prompts/company-agent.uk.ts`

Оновити опис теми 1 «Посада»:

- Зібрати: назва ролі/посади; за потреби нейтрально — досвід, грейд або рівень відповідальності (якщо HR не вказав); контекст команди.
- Прибрати обов’язкове формулювання `junior/middle/senior` як частину теми.

Оновити правила діалогу:

- Порожня історія → привітання + **одне** питання лише про назву посади/роль.
- Не починати діалог із питання про junior/middle/senior і не нав’язувати IT-грейди, якщо HR їх сам не використав.
- Якщо після відповіді про роль бракує інформації про досвід/відповідальність — одне нейтральне уточнення; потім команда (якщо ще не зібрана).

Інші теми (вимоги, очікування, умови роботи) і формат `READY:true|false` — без змін.

### 2. `backend/src/agents/company-agent.test.ts`

Додати/оновити assert(и), що промпт:

- не вимагає junior/middle/senior як обов’язковий елемент теми «Посада»;
- містить правило: перше питання лише про роль;
- містить правило нейтрального уточнення досвіду/відповідальності лише за потреби.

## Критерії готовності

- [ ] У промпті немає обов’язкового `junior/middle/senior` у темі «Посада».
- [ ] Є явні інструкції: перше питання = роль; нейтральне уточнення рівня лише якщо бракує.
- [ ] Тести `company-agent.test.ts` проходять.
- [ ] Поведінка інших тем і READY-логіки не змінена.

## Ризики

- LLM може інколи все одно згадати junior/middle/senior — пом’якшується явною забороною в промпті; без hardcoded opening це прийнятний компроміс підходу A.
