КОРОТКИЙ ПІДСУМОК

Сьогодні закрито День 1 з 22-денного плану розробки MVP: з нуля піднято
monorepo, спроєктовано й застосовано схему БД, налаштовано локальне середовище
(Docker + PostgreSQL + Prisma), реалізовано мінімальний runtime (Vite + Vue 3 +
Express + TypeScript) і верифіковано end-to-end сценарій запуску. У репозиторії
за день зроблено ~30 комітів.

ПРИЙНЯТІ РІШЕННЯ
================================================================================

АРХІТЕКТУРНІ

1. Monorepo npm workspaces — один npm install, єдині кореневі команди для обох
   пакетів.
2. Цільовий стек одразу (Vite + Vue 3 + Express + TypeScript), без тимчасового
   placeholder scaffold.
3. Нормалізована схема БД замість універсальних таблиць — окремі сутності для
   prep/live, company/candidate profiles; jsonb лише для змінних структур
   (наприклад, плюси/ризики звіту).
4. Існуючі JS-модулі збережено (healthcheck.js, hr-user.js) — backend TypeScript
   імпортує їх через allowJs, без переписування працюючого коду.
5. Один endpoint /api/health для Day 1 — мінімальний контракт для UI і curl.
6. HTTP 200 навіть при помилках (ok: false у body) — зручніше для статус-сторінки,
   ніж обробка 503.

ІНФРАСТРУКТУРНІ

7. PostgreSQL 16 у Docker Compose — параметризований healthcheck через env vars.
8. Prisma v7 — datasource config і defaults узгоджені з новою версією.
9. Seed через upsert — ідемпотентність, повторний запуск не створює дублікатів.
10. SHA-256 для пароля seed — тимчасово для локальної розробки (auth API ще не
    реалізовано).

ПРОЦЕСНІ

11. Design spec → implementation plan → код — кожен блок спочатку задокументований,
    потім реалізований.
12. Вертикальний зріз Day 1 — від структури репо до працюючої демонстрації в
    браузері.
