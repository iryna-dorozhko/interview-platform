Додано implementation plan у docs/superpowers/plans/.


2. BACKEND — ПІДТВЕРДЖЕННЯ ПРОФІЛЮ

- **Маршрут** `POST /api/prep/:interviewId/confirm`:
    • перевірка власника співбесіди (`requireAuth` + `requireHr`);
    • 404 якщо профіль ще не згенеровано (`finish` не викликано);
    • 409 якщо `confirmedAt` уже встановлено;
    • встановлює `confirmedAt: new Date()` на `CompanyProfile`;
    • якщо `Interview.status === DRAFT` → переводить у `AWAITING_CANDIDATE`;
    • повертає `{ profile, interviewStatus }`.
- **Захист від скидання:** `DELETE /api/prep/:interviewId` повертає 409
  «Profile is confirmed and cannot be reset», якщо профіль підтверджено.
- **GET /prep/:interviewId** включає `confirmedAt` у відповідь профілю.
- Unit-тести в `prep.test.ts`: confirm success, 404, 409, 403, delete blocked.


3. FRONTEND — UI ПІДТВЕРДЖЕННЯ

- API-клієнт `confirmPrepProfile()` у `frontend/src/api/prep.ts`.
- `CompanyPrepView.vue`:
    • кнопка «Підтвердити профіль» з діалогом підтвердження;
    • після confirm — банер «✓ Підтверджено <дата>»;
    • кнопка «Видалити чат» disabled після підтвердження;
    • індикатор завантаження (`confirming`).


4. АВТОМАТИЗАЦІЯ ВЕРИФІКАЦІЇ

- Скрипт `scripts/screenshot-prep.mjs`:
    • через API проходить повну анкету (5 обмінів + finish);
    • через Playwright робить скріншоти профілю до/після confirm;
    • перевіряє українську мову відповідей агента.

# ДЕНЬ 8 — СТВОРЕННЯ СПІВБЕСІДИ

================================================================================
ЩО БУЛО ЗРОБЛЕНО (ДЕНЬ 8)
================================================================================

1. ПРОЄКТУВАННЯ (DESIGN SPECS)

Написано й затверджено дизайн-документ у docs/superpowers/specs/:

Додано implementation plan у docs/superpowers/plans/.


2. BACKEND — ГЕНЕРАЦІЯ КОДУ ТА СТВОРЕННЯ СПІВБЕСІДИ

- **Утиліта** `backend/src/utils/joinCode.ts`:
    • 6 символів з алфавіту `23456789ABCDEFGHJKLMNPQRSTUVWXYZ`;
    • без `0`, `O`, `1`, `I` — зручно диктувати голосом;
    • `crypto.randomInt()` для криптографічно стійкої випадковості.
- Unit-тести `joinCode.test.ts`: довжина, алфавіт, варіативність.
- **Маршрут** `POST /api/interviews`:
    • створює `Interview` з `status: DRAFT` і згенерованим `joinCode`;
    • retry до 5 разів при колізії unique-constraint (P2002);
    • 500 лише якщо вичерпано всі спроби.
- Unit-тести в `interviews.test.ts` для create endpoint.


3. FRONTEND — КНОПКА ТА БАНЕР КОДУ

- API-клієнт `createInterview()` у `frontend/src/api/interviews.ts`.
- `HomeView.vue`:
    • кнопка «Створити співбесіду» поруч із «Анкета компанії»;
    • зелений банер `.created-banner` з 6-символьним кодом;
    • кнопка «Перейти до анкети →» → `/prep/:interviewId`;
    • обробка помилок і стан завантаження.


4. АВТОМАТИЗАЦІЯ ВЕРИФІКАЦІЇ

    • скріншот банера та переходу в prep-чат.
